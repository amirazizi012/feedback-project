# پروژه سیستم رسیدگی به بازخورد (Feedback Board)

## راهنمای اجرای لوکال (Local Setup) روی سیستم شما

برای اجرا کردن پروژه روی سیستم شخصی خود، گام‌های زیر را در ترمینال خود (سیستم‌عامل ویندوز، لینوکس یا مک) اجرا کنید.

**گام ۱: دریافتِ سورس‌کد و ورود به مسیر پروژه**

<div dir="ltr">

```bash
git clone https://github.com/your-username/feedback-board.git
cd feedback-board
```

</div>

**گام ۲: ایجاد و فعال‌سازی محیط مجازی پایتون (Virtual Environment)**
این کار باعث می‌شود پکیج‌های این پروژه با سایر پروژه‌های سیستم شما تداخل پیدا نکنند.

<div dir="ltr">

```bash
python -m venv env

# در ویندوز:
env\Scripts\activate

# در مک یا لینوکس:
source env/bin/activate
```

</div>

**گام ۳: نصب نیازمندی‌های دیتابیس و فرم‌ورک**

<div dir="ltr">

```bash
pip install -r requirements.txt
```

</div>

**گام ۴: اجرای سرور (Uvicorn / FastAPI)**

<div dir="ltr">

```bash
uvicorn app.main:app --reload
```

</div>

اگر این دستور با موفقیت اجرا شود، برنامه روی پورت `8000` قرار می‌گیرد. 
- برای دیدن سایت اصلی به `http://127.0.0.1:8000/` بروید.
- برای دیدن API Docs خودکار (Swagger UI) به `http://127.0.0.1:8000/docs` مراجعه کنید.



## اطلاعات ورود به پنل مدیریت

## راهنمای اجرای سریع (Quick Start)

برای راحتی کار، فایل‌های راه‌انداز خودکار اضافه شده است:

**در ویندوز (Windows):**
فقط کافیست روی فایل `run.bat` دو بار کلیک کنید. (خودکار محیط مجازی ساخته، نیازمندی‌ها را نصب و سرور را اجرا می‌کند).

**در لینوکس یا مک (Linux/Mac):**
دستور زیر را در ترمینال اجرا کنید:
```bash
chmod +x run.sh && ./run.sh
```

**اجرا با داکر (Docker):**
```bash
docker build -t feedback-app .
docker run -p 8000:8000 feedback-app
```

### اطلاعات ادمین برای لاگین در داشبورد
یک کاربر اولیه ادمین زمان اولین راه‌اندازی، خودکار برای شما ساخته می‌شود:
- **نام کاربری:** `admin`
- **رمز عبور:** `password123`

---
<div dir="ltr">
در پیاده‌سازی این پروژه از رویکردهای زیر استفاده شده است:

1.  **FastAPI:** انتخاب شد به دلیل سرعت بسیار بالا (Asynchronous) و تولید خودکار مستندات API (Swagger) که سرعت توسعه را دوچندان می‌کند.
2.  **معماری ماژولار (Routers & Dependencies):** کدها به لایه‌های مجزا (دیتابیس، احراز هویت، منطق تجاری) تقسیم شدند تا نگهداری و تست پروژه در مقیاس بزرگ آسان باشد.
3.  **SQLAlchemy (ORM):** استفاده از ORM باعث می‌شود برنامه به یک دیتابیس خاص (مثل SQLite) وابسته نباشد و به راحتی بتوان آن را به PostgreSQL یا MySQL متصل کرد.
4.  **Bcrypt & JWT:** برای امنیت از هش‌کردن پسوردها با کتابخانه `bcrypt` و برای مدیریت نشست‌های کاربر از توکن‌های استاندارد `JWT` استفاده شده است.
5.  **Tailwind CSS:** برای فرانت‌اند از یک رویکرد مدرن استفاده شده تا بدون نیاز به فریم‌ورک‌های سنگین جاوااسکریپت، ظاهر برنامه کاملاً زیبا و ری‌اسپانسیو باشد.
</div>
