

# پروژه سیستم رسیدگی به بازخورد (Feedback Board)

## راهنمای اجرای لوکال (Local Setup) روی سیستم شما

برای اجرا کردن پروژه روی سیستم شخصی خود، گام‌های زیر را در ترمینال خود (سیستم‌عامل ویندوز، لینوکس یا مک) اجرا کنید.

**گام ۱: دریافتِ سورس‌کد و ورود به مسیر پروژه**

<div dir="ltr">

```bash
git clone https://github.com/your-username/feedback-board.git
cd feedback-board
```

</div>

**گام ۲: ایجاد و فعال‌سازی محیط مجازی پایتون (Virtual Environment)**
این کار باعث می‌شود پکیج‌های این پروژه با سایر پروژه‌های سیستم شما تداخل پیدا نکنند.

<div dir="ltr">

```bash
python -m venv env

# در ویندوز:
env\Scripts\activate

# در مک یا لینوکس:
source env/bin/activate
```

</div>

**گام ۳: نصب نیازمندی‌های دیتابیس و فرم‌ورک**

<div dir="ltr">

```bash
pip install -r requirements.txt
```

</div>

**گام ۴: اجرای سرور (Uvicorn / FastAPI)**

<div dir="ltr">

```bash
uvicorn app.main:app --reload
```

</div>

اگر این دستور با موفقیت اجرا شود، برنامه روی پورت `8000` قرار می‌گیرد. 
- برای دیدن سایت اصلی به `http://127.0.0.1:8000/` بروید.
- برای دیدن API Docs خودکار (Swagger UI) به `http://127.0.0.1:8000/docs` مراجعه کنید.

## راهنمای اجرای سریع (Quick Start)

برای راحتی کار، فایل‌های راه‌انداز خودکار اضافه شده است:

**در ویندوز (Windows):**
فقط کافیست روی فایل `run.bat` دو بار کلیک کنید. (خودکار محیط مجازی ساخته، نیازمندی‌ها را نصب و سرور را اجرا می‌کند).

**در لینوکس یا مک (Linux/Mac):**
دستور زیر را در ترمینال اجرا کنید:
```bash
chmod +x run.sh && ./run.sh
```

**اجرا با داکر (Docker):**
```bash
docker build -t feedback-app .
docker run -p 8000:8000 feedback-app
```

## توضیحاتی درباره تصمیم‌های فنی که گرفتم

بر اساس الزامات ذکر شده در صورت تمرین، سعی کردم تمام موارد را به ساده‌ترین اما مقیاس‌پذیرترین شکل ممکن پیش ببرم:

۱. **استفاده از FastAPI به جای Flask/Django:** به دلیل پشتیبانی عالی از `Type Hinting` در پایتون ۳ و سرعت بالای پردازش اطلاعات، این فریم‌ورک باعث می‌شود کدهای بسیار تمیزتر و API های مدرن‌تری داشته باشیم.
۲. **مدل‌سازی دیتابیس با SQLAlchemy:** به دلیل اینکه باید قابلیت استفاده از MySQL یا `PostgreSQL` را داشته باشیم اما در زمان اجرای تستی با مشکل پیچیدگی کانفیگ مواجه نشویم، من از مدل‌سازی استاندارد ORM استفاده کردم. پروژه الان روی SQLite اجرا می‌شود اما با تغییر خط مربوط به کانکشن در `database.py`، بدون حتی تک‌خط تغییر در کدها، تمام اطلاعات به یک دیتابیس قدرتمند کشیده می‌شود و ارتباط بین موجودیت‌ها نیز لحاظ شده است.
۳. **پرهیز از Over-engineering در فرانت‌اند:** در نیازمندی‌ها ذکر شده بود استفاده از HTML/CSS کفایت می‌کند و نیازی به JS سنگین نیست. من صفحات را به طور کامل درون پوشه `templates` به صورت استاتیک و با استفاده از فریم‌ورک کلاس‌بندی `Tailwind CSS` کاملاً ری‌اسپانسیو (Responsive) درآوردم که هم ظاهری تمیز دارد و هم بار اضافه‌تری را به سیستم وارد نمی‌کند. احراز هویتِ سمت ظاهر سیستم با استفاده از جاوااسکریپت پایه (`Vanilla JS`) پیاده‌سازی شده که به راحتی برای نشان‌دادن داشبورد از توکن صادر شدهٔ `JWT` استفاده می‌کند.
